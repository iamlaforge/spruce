# Spruce Context

*This file tells Spruce about your project so every command can calibrate its work to what you're actually making. Replace each prompt below with your answer. Keep what's useful, delete what isn't.*

*This is the foundational file in the Spruce workflow:*

```
/spruce up  →  Discovery (HCD)  →  Generative  →  Diagnostic  →  Corrective  →  /finish
```

*After this file is created, the Discovery tier (`/personas`, `/jtbd`, `/journey`) deepens the user grounding into structured artifacts that every downstream command calibrates against.*

## Product

What is the product, and what does it do?

## Audience

*The starting capture of who the product serves. Once `/personas` runs, this section becomes a brief pointer to `.personas.md` — that file holds the structured persona work (motivations, fears, contexts of use, design implications) that downstream commands use as the canonical audience source.*

Who uses it? What's their expertise level and context?

## Character

What should the product feel like? A few words or a short sentence.

## Density

How dense should the interface be — spacious, balanced, or dense?

## Voice

What should the product sound like? Direct and precise, warm, formal, playful, something else?

## Platform

What platforms does this run on?

## Dark mode

Does the product support dark mode, or light mode only?

---

## Typography preferences

Any typefaces you want to use, avoid, or must use for brand reasons?

## Color preferences

Any colors that must be used (brand colors) or avoided? Palette direction?

## What this should NOT feel like

Anything specific you want to avoid — aesthetics, registers, comparisons to other products?

## UX patterns to always use or avoid

Any interaction patterns you always want or never want?

---

## When Spruce has to decide

These apply when you ask Spruce to "decide for me" during `/decide`. Each is optional — answer only the ones you have a preference on.

**Restraint or expression?** When choosing between quiet confidence and bold commitment, which way should Spruce lean?

**Classic or distinctive typography?** When choosing between proven, readable choices and character-forward, unusual ones, which way should Spruce lean?

**Symmetry or tension in layout?** When choosing between balanced structure and asymmetric dynamism, which way should Spruce lean?

---

## Notes

Anything else Spruce should know — brand guidelines, constraints, things that have been tried and rejected, historical decisions?
